# Echo 的产品设计：不只是聊天

如果只是做一个聊天 Demo，Echo 不需要这么复杂。

但我想做的不是一个只有输入框和消息列表的页面，而是一个更接近移动端 App 的即时通讯产品。所以在基础聊天之外，我给它加了动态、星域、关系页、表情包、背景设置、定时发送和引力圈。

这篇主要记录 Echo 的产品设计。

项目地址：https://github.com/ry520-stack/-Echo-IM

## 整体风格：极简、大量留白、移动端优先

Echo 的界面方向是极简。

我不太想把聊天界面做成按钮很多、信息很满的样子。聊天软件最核心的东西还是消息本身，所以界面里保留了大量留白，按钮尽量收起来，很多操作通过长按、滑动、弹层完成。

这样做的好处是：

- 第一眼更干净
- 移动端不容易拥挤
- 聊天消息更突出
- 后续加功能时不至于把页面塞满

当然这也有代价。比如功能入口必须设计清楚，否则用户会找不到。所以 Echo 后面把一些概念重新整理了一遍：

- 聊天：即时沟通
- 动态：发布内容
- 星域：好友分组和权限
- 关系星域：我和某个好友之间的关系页
- 群星域：群聊的资料和关系页

这样比到处都叫“星轨”更清楚。

## 星域：好友分组，也是权限系统

Echo 里“星域”不是单纯的好友分组。

它同时承担动态权限的作用。

比如我可以创建：

- 家人星域
- 同学星域
- 同事星域
- 密友星域

发布动态时，可以选择哪些星域可见，也可以单独排除某些人。

最终权限可以理解成：

```text
最终可见范围 = 允许的星域 - 单独排除的人
```

这比普通的“谁可见/谁不可见”更像 Echo 自己的体系。

好友可以属于多个星域。比如一个人既是“同学”，也是“密友”。关系页里可以选择展示其中一个主要星域，后面也可以继续扩展成更完整的关系管理。

## 关系星域：聊天页左滑进入

Echo 的私聊页可以左滑进入“关系星域”。

这个页面不是朋友圈，也不是普通资料页，而是展示“我和这个人之间的关系”。

里面包括：

- 对方头像
- 昵称
- 在线状态
- Echo ID
- 所属星域
- 认识时间
- 最近互动
- 关系数据
- 对方最近动态

如果对方没有发过动态，就显示“暂无动态”。

我不想把所有设置都塞在聊天页里，所以资料设置、星域设置、动态权限、聊天背景这些东西会放到好友资料页里。聊天页只负责聊天，关系星域负责关系。

## 动态：悬浮折叠和宫格展示

Echo 的动态系统支持图片发布。

目前单条动态最多 18 张图，展示方式有两种：

第一种是悬浮折叠。

图片会像卡片一样堆叠起来，用户可以滑动切换。这个展示方式更有视觉感，适合少量图片。

第二种是宫格展示。

一页最多 9 张，超过 9 张时分页。这样发很多图时不会横向一张张找，浏览效率更高。

发布动态时，上传图片也改成了宫格预览，而不是横向长条。因为在手机上横向找图真的很慢，尤其是图片多的时候。

## 表情包：默认 4 个，展开 12 个

表情包也做了移动端适配。

默认情况下，只展示 4 个，避免挤占聊天空间。

用户手动展开后，一页展示 12 个。表情多的时候可以左右分页，不需要一直横向慢慢滑。

这个设计来自一个很实际的问题：如果只做横向滚动，表情包数量一多，找表情会很慢。分页宫格更接近常见聊天 App 的体验。

## 引力圈：重要联系人的快速入口

引力圈是 Echo 里一个偏概念化的功能。

它不是会话列表，也不是通讯录，而是一个放重要联系人的区域。

可以把某个好友“捕捉”进引力圈。这样这个人就从普通会话列表里被单独拿出来，成为一个更重要的关系入口。

这个功能的意义不是炫技，而是给关系做分层：

- 会话列表：所有聊天
- 通讯录：所有好友
- 引力圈：最重要的人

它让 Echo 不只是一个消息工具，也有一点关系管理的味道。

## 定时发送和回声胶囊

Echo 里还设计了“回声胶囊”。

它对应的是定时消息。

比如你可以提前写一条消息，在某个时间发送出去。这个功能看起来不大，但对聊天软件来说很有想象空间：

- 生日祝福
- 纪念日提醒
- 给未来的自己留言
- 定时发送一段话

目前项目里已经有定时消息入口和后端接口，后续还可以继续做成更完整的“时间胶囊”体验。

## 背景设置：自己的背景自己控制

Echo 支持多种背景：

- 会话列表背景
- 聊天界面背景
- 引力圈背景
- 好友专属聊天背景

这里有一个原则：背景应该是自己的设置，不能同步影响对方。

聊天软件里的背景是个人体验的一部分，不应该因为对方设置了某张图，我这边也被改掉。

所以后面我把背景设置按用户、页面和好友关系拆开处理，避免相互污染。

## 群聊和群星域

Echo 也加了群聊。

目前支持：

- 创建群聊
- 拉好友进群
- 群头像
- 群名称
- 群公告
- 群成员
- 群主、管理员、成员
- 我在本群的昵称和备注
- 群主转让、退群、解散群

群聊也有左滑页，叫“群星域”。

私聊左滑是关系星域，群聊左滑就是群星域。这样产品逻辑会更统一。

## 总结

Echo 的重点不只是“能聊天”。

我更想把它做成一个有自己产品逻辑的 IM：

- 聊天解决即时沟通
- 动态解决内容表达
- 星域解决关系圈层和权限
- 关系星域解决人与人之间的关系展示
- 引力圈解决重要关系入口
- 表情包、背景、定时发送提升移动端体验

它现在还在迭代，但方向已经比较清楚：不是堆功能，而是让每个功能都服务于“关系”和“沟通”。

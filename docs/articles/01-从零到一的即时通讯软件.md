# Echo 篇 1：一个云计算学生，从零到一的即时通讯软件

## 起点

我是云计算专业的大二学生，备考 HCIE-Cloud 认证。做 Echo 的初衷很务实——上一个项目 StudyTracker 是为了简历有东西写。但打卡软件确实没什么用，我想做一个**自己能用、能发给家人用**的东西。

Echo 就是这么来的。

## 技术选型

```
前端: React 19 + TypeScript + Tailwind CSS 3 + Vite 8 + Framer Motion 12
后端: Node.js + Express + Prisma + PostgreSQL 16
实时: Socket.IO (WebSocket + polling 降级)
部署: Docker Compose 三容器 + Nginx 反向代理 + 阿里云 ECS
```

这不是"学来的技术栈"，是**用出来的**。每个技术都是因为需求才加的——WebSocket 是因为需要实时聊天，Framer Motion 是因为 CSS transition 做不出物理阻尼感，Docker 是因为要部署到云服务器。

## 架构

三个 Docker 容器：
- `echo-frontend`: Nginx 托管 React 静态文件 (8080→80)
- `echo-backend`: Express API 服务 (9090→3001)
- `echo-db`: PostgreSQL 16 Alpine (5433→5432)

前端是 SPA 单页应用，HashRouter 路由，所有页面共享一个 WebSocket 连接。

## 核心功能

第一个版本就包含了注册登录、一对一聊天、好友系统、群组、朋友圈。这些是基础骨架。真正让 Echo 不一样的，是后面加的那些东西——引力圈、对称隐私、流体语音、悬浮堆叠画廊。

## 教训

**不要一开始就追求完美。** 先跑起来，再迭代。Echo 的第一个版本连样式都丑得要命，但功能是通的。后面几十轮迭代才改到今天的样子。

# Echo 关系星域与侧边栏整理方案

本文档记录当前已经落地的关系星域、动态权限和侧边栏设计，不再保留过期的阶段性 TODO。

## 当前产品逻辑

- 聊天：即时沟通主界面。
- 动态：用户发布内容，支持公开、仅自己、部分可见、不给谁看。
- 星域：好友分组，也是动态可见权限的基础。
- 关系星域：从聊天页左滑进入，展示你和某个好友之间的资料、关系数据和最近动态。
- 好友设置：从头像/昵称进入，管理备注、专属背景、星域归属等关系设置。

## 已落地功能

- 左滑关系星域页：标题为“与 {peerName} 的星域”，无动态时显示“暂无动态”。
- 星域管理页：支持新建、编辑、删除星域，选择颜色，添加/移除好友。
- 动态权限：支持星域允许和单独排除好友，后端按权限过滤动态。
- 后端权限校验：发布动态时校验星域归属和好友关系，星域加人时只能添加已通过好友。
- 原生弹窗清理：动态草稿确认、删除确认等使用应用内弹窗，不使用浏览器原生 confirm。
- 旧命名清理：对用户可见位置统一使用“动态”。

## 侧边栏改造目标

当前侧边栏的问题：

- 入口像普通卡片菜单，和 Echo 的“聊天 + 星域 + 动态”体系不够统一。
- 设置入口藏在底部小齿轮里，手机上不够直接。
- 星域管理入口不够明显，但它现在已经是权限系统核心。
- 绿点动画和多块渐变卡片显得轻，缺少成熟 App 的稳定感。

改造方向：

- 改成“Echo 控制面板”式侧栏，而不是普通功能卡片。
- 顶部展示用户身份、Echo ID、状态签名，在线点保持安静。
- 中部使用清爽列表入口，突出核心路径：
  - 通讯录
  - 动态
  - 星域
  - 回声胶囊
- 底部直接展示设置、深色模式、退出登录，不再折叠到小齿轮菜单。
- 使用 lucide 图标，少用 emoji，保持应用感。

## 侧边栏信息架构

顶部身份区：

- 头像
- 昵称
- Echo ID
- 个性签名，点击可编辑
- 在线状态点

主导航：

- 通讯录：好友、请求、群组入口
- 动态：好友近况、可见星域内容
- 星域：分组和动态权限管理
- 回声胶囊：定时消息和私密内容入口

底部操作：

- 深色模式切换
- 设置
- 退出登录

## 实现约束

- 不改路由结构，只复用现有路径：`/friends`、`/moments`、`/star-zones`、`/time-capsule`、`/settings`。
- 不改后端接口。
- 不引入新依赖，使用现有 `lucide-react`、`framer-motion`、Tailwind。
- 改完必须跑前端构建。
- 部署只重建前端容器，不碰数据库、uploads、volume。

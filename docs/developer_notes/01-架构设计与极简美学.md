# Echo 稀土掘金篇 1：架构设计与星际 UI 体系

> 作者：ry520-stack | 一个云计算专业大二学生的全栈独立开发实践

## 引言

Echo 是一款以"宇宙、引力、星轨"为核心设计语言的即时通讯应用。本文作为系列第一篇，将拆解 Echo 的整体架构设计、技术选型以及独特的"星际 UI"视觉体系。

## 技术栈

```
前端: React 19 + TypeScript + Tailwind CSS 3 + Vite 8 + Framer Motion 12
后端: Node.js + Express + Prisma + PostgreSQL 16 (Alpine)
实时: Socket.IO (WebSocket + polling fallback)
部署: Docker Compose 三容器 + Nginx 反向代理 + 阿里云 ECS
```

## 核心设计语言：星际与共鸣

### 色彩体系
- 品牌色: `#6366f1` (靛蓝) → `#a855f7` (紫罗兰)
- 深空黑: `#05050A` (OLED 纯黑)
- 浅色基底: `#f9fafb` (冷灰白)

### 材质体系
- Glassmorphism 毛玻璃: `backdrop-blur-xl bg-white/80 dark:bg-gray-900/80`
- 弹簧物理动效: Framer Motion `spring stiffness:150-200 damping:20-25`

### 三大核心页面
1. **引力圈 (Gravity Zone)** — 深空纯黑背景 + 轨道环装饰 + 星球呼吸动画
2. **星轨 (Orbit)** — 朋友圈动态 + 悬浮堆叠画廊
3. **回声胶囊 (Time Capsule)** — 定时消息 + 私密日记

## 架构设计

### 前端路由
```
/              → 会话列表 (ChatPage)
/chat/:id      → 聊天窗口
/chat/:id/settings → 好友设置
/friends       → 好友管理
/moments       → 星轨 (朋友圈)
/time-capsule  → 回声胶囊
/settings      → 个人设置
/favorites     → 引力圈 (Gravity Zone)
```

### 后端 API
- REST API: Express + Prisma + JWT 认证
- WebSocket: Socket.IO 实时消息推送
- 文件上传: Multer 处理图片/语音/表情

### Docker 三容器
- `echo-frontend`: Nginx 静态服务 (8080:80)
- `echo-backend`: Express API (9090:3001)
- `echo-db`: PostgreSQL 16 Alpine (5433:5432)

## 关键工程决策

1. **为什么用 Framer Motion 而不是纯 CSS？** 弹簧物理模型提供真实的阻尼感，CSS transition 无法模拟
2. **为什么用 SVG feColorMatrix 做流体效果？** 纯 CSS border-radius 无法实现粘滞拉丝感
3. **为什么图片要前端压缩？** 手机原图 8MB+ 直接上传会 OOM，压缩到 200KB 后流畅

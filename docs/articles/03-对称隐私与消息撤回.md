# Echo 篇 3：对称隐私与消息撤回

## 对称隐私

传统聊天软件的已读回执是单向的——我开了你就能看到我读了，我不开你就看不到。但这不够尊重隐私。

Echo 引入**对称隐私**：

```typescript
const isRead = (msg: Message) => {
  if (!readReceiptOn) return false;           // 我没开启
  if (peer?.readReceiptsEnabled === false) return false; // 对方没开启
  return msg.readReceipts?.some(r => r.userId === peerId);
};
```

**只要任意一方关闭已读反馈 → 双方所有消息都隐藏已读状态。** 要么都坦诚，要么都自由。

技术上需要服务端支持。在 User 模型新增 `readReceiptsEnabled` 字段，搜索 API 返回该字段，前端判断。

## 消息本地删除

删除仅本地执行，不调后端 API，不影响对方：

```typescript
setMessages(prev => prev.filter(msg => msg.id !== m.msgId));
```

## 120秒撤回

```typescript
const within2min = (Date.now() - new Date(m.createdAt).getTime()) < 120000;
if (m.isMine && within2min) {
  await api('PUT', `/api/messages/${msgId}/recall`);
}
```

撤回后双方气泡替换为"撤回了一条消息"。后端广播 WebSocket 事件同步更新。

## 消息长按菜单

长按消息气泡 500ms → 弹出上下文菜单（引用 / 删除 / 撤回（<120s））。删除选项仅本地生效，撤回选项仅 2 分钟内显示。

## 在线状态同步

会话列表和聊天室共用同一个在线状态池，统一由 `SocketContext` 管理。收到 `message:receive` 或 `typing:update` 事件时强制将该用户设为在线。

```typescript
s.on('message:receive', forceOnline);
s.on('typing:update', forceOnline);
```

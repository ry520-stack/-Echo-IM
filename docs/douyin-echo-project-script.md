# 抖音项目介绍文案：Echo IM

这是我做的一个云端即时通讯项目，名字叫 Echo。

它不是普通聊天 Demo，而是一个完整部署在云服务器上的 IM 应用。前端用 React，后端用 Node.js，实时通信走 Socket.IO，数据库用 PostgreSQL，生产环境通过 Docker Compose 部署在 ECS 上，并绑定了 HTTPS 域名 `echo-im.cloud`。

Echo 支持好友聊天、群聊、拉群、语音消息、语音通话、聊天记录搜索、会话列表搜索、朋友圈动态、图片上传、表情包管理、自定义背景、自定义来电铃声、已读回执、在线状态，还有双人共同宠物。

项目里比较有特色的是这个“共同宠物”。它不是悬浮在屏幕上的装饰，而是像第三个聊天成员一样，偶尔在两个人的聊天里主动发一句话。后续还可以接入大模型，让宠物根据少量上下文生成更自然的回复。

从云计算角度看，这个项目覆盖了很多真实上线问题：域名解析、HTTPS、Nginx 反向代理、WebSocket、WebRTC、文件上传、数据库持久化、容器部署、移动端 WebView 适配、麦克风权限、语音通话信令、上传文件存储和接口安全。

它的架构是这样的：

用户通过 HTTPS 访问 Echo，Nginx 负责入口转发，前端容器提供页面，后端容器提供 API 和 Socket.IO 服务，PostgreSQL 保存用户、好友、群聊、消息和朋友圈数据，上传的图片、语音、表情包、铃声保存到服务器持久化目录。

如果继续升级，我会把文件迁移到对象存储，接 CDN 加速，再加 TURN 服务提升语音通话成功率，最后用 Redis 支持多实例在线状态和消息扩展。

这个项目最适合作为云计算方向的综合作品，因为它不只是写页面，而是从产品功能、后端接口、实时通信到云端部署完整打通。

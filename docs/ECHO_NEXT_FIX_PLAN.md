# Echo 下一轮修复计划

更新时间：2026-05-23

## 1. 设置页里的“服务器地址”

当前“服务器地址”用于指定前端请求后端 API 的地址，例如 ECS 的 `http://8.140.194.214:9090` 或域名后的 API 地址。

它不是数据库密码，也不是服务器登录密码。IP 地址本身不是机密，别人知道 IP 不代表能登录服务器。

但这里有几个风险要处理：

- 当前如果使用 `http://`，登录 token 和接口数据在网络层不是端到端加密，公共 Wi-Fi 下不够安全。
- 普通用户不应该看到或修改“服务器地址”，否则可能误填导致无法登录，或者被诱导改到假服务器。
- 等正式域名下来后，应固定为 HTTPS API 地址，普通设置页隐藏这个输入，只保留开发调试入口。

建议：

- 开发者模式才显示“服务器地址”。
- 普通用户设置页不显示该项。
- 正式环境使用 HTTPS 域名，不再让用户手动输入 ECS IP。
- 前端只允许同源或白名单 API 地址，避免被改到恶意后端。

## 2. “与 Shoot 的星域”页面问题

截图页面整体方向是对的，但现在数据空得太多，体验还不完整。

### 需要修复的问题

- “资料设置”按钮点击没有反应。
- 资料设置应该进入好友设置页，也就是以前聊天页点击好友名字进入的页面。
- 聊天页点击好友名字进入设置要关闭。
- 好友设置页的唯一入口改成：聊天页左滑进入“关系星域” -> 点击“资料设置”。
- 从好友设置页返回后，应回到“与 XXX 的星域”，不是回聊天页或会话列表。

### 关系数据区调整

当前四个块：

- 所属星域
- 认识时间
- 最近互动
- 共同图片

需要改成更有用的四个块：

- 所属星域
- 认识时间
- 回声值
- 最近连接

说明：

- “共同图片”不要了。
- “共同图片”位置改成“回声值”，但不要叫“亲密度”，避免像 QQ。
- 回声值由聊天、点赞动态、评论动态、互相访问、语音/通话等互动累积。
- 后续需要有一个独立页面查看“我的好友回声值排行”。

## 3. 所属星域逻辑

一个好友可以属于多个星域，例如：

- 家人星域
- 至亲星域
- 同学星域

但“关系星域”页只展示一个主星域。

规则建议：

- 每个好友允许设置一个"所属星域"作为"关系星域"页面"所属星域"展示的“星域”。
- 如果未设置，默认取该好友所属星域里的第一个。
- 如果没有任何星域，显示“暂无”。
- 点击“展示星域”卡片，打开自定义弹窗选择一个星域。
- 弹窗不能用浏览器原生弹窗，要用项目自己的 Modal 样式。

后端需要保存：

- `Friend.displayGroupId` 或单独的关系设置表字段。

## 4. 认识时间逻辑

现在“认识时间”显示暂无，不够好。

需要改成：

- 默认显示软件内互相加好友成功的日期，格式：`2026年5月23日`。
- 点击“认识时间”卡片后，切换显示“现实相识时间”。
- 现实相识时间可以在资料设置页里手动设置。
- 如果没有设置现实相识时间，显示“未设置现实相识时间”。

后端需要保存：

- 好友关系创建/接受时间：可用现有 `Friend.createdAt`，但最好确认是否表示接受时间。
- 现实相识时间：建议字段 `Friend.realMetAt`。

## 5. 回声值

“亲密度”这个名字不要用。

推荐命名：

- 页面展示：`回声值`
- 排行页：`回声排行`
- 说明文案：`由聊天、动态互动和通话等行为累积`

初始实现可以先做 MVP：

- 每发送一条消息 +1
- 每点赞对方动态 +2
- 每评论对方动态 +3
- 每次语音/通话完成 +5
- 每天最多计入一定上限，避免刷分

需要新增：

- 后端统计服务，例如 `relationshipScore.service.ts`
- 前端好友回声排行页
- 关系星域页展示当前好友的回声值

## 6. 最近互动 / 最近连接

## 8. 好友资料入口统一

最终入口设计：

- 聊天页顶部点击好友名字：不进入设置。
- 聊天页左滑：进入“与 XXX 的星域”。
- 关系星域页点击“资料设置”：进入好友设置页。
- 好友设置页返回：回到关系星域页。

这样聊天页不塞太多入口，结构更清楚。
